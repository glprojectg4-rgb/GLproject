1. Introduction

The "Blood Donation Management System" is a web-based application designed to facilitate the complex process of managing blood donations, stock levels, and donor communications. In the context of healthcare, efficiency and clarity are paramount. This project aims to digitize manual processes, reducing errors and saving time for medical staff.

The objective of this Human-Computer Interaction (HCI) project is to design an interface that is not only functional but also ergonomic, intuitive, and accessible. The system caters to three distinct user roles: Administrators, Managers, and Agents, each with specific needs and workflows.

2. Analysis

2.1 Users and Tasks

We identified three primary user categories:

  • Agents: Front-line staff responsible for registering donations, checking stock, and contacting donors. Their work is fast-paced, requiring quick data entry and clear visual feedback.
  • Managers: Operational supervisors who oversee donor lists, handle communications, and manage staff accounts. They require overview dashboards and easy access to reports.
  • Administrators: System maintainers responsible for global settings, disease lists, and system alerts. Their interface focuses on control and configuration.

2.2 Context of Use

The system is used in a clinical or office environment, likely on desktop computers. Users may be multitasking, so the interface must minimize cognitive load and prevent errors, especially when handling sensitive donor data.

3. Architecture

3.1 Chosen Model: MVC (Model-View-Controller)

For the architectural design of this application, we selected the MVC (Model-View-Controller) pattern. This choice is justified by the need to separate the user interface (View) from the business logic (Controller) and the data structure (Model).

  • Model: Represents the database tables (Users, Donors, Stocks). It handles data integrity.
  • View: The HTML/CSS pages presented to the user. In our project, we focused heavily on this layer to ensure a premium User Experience (UX).
  • Controller: The PHP scripts that process user inputs (logins, form submissions) and update the Model or View accordingly.

This separation allows for easier maintenance and ensures that UI changes (like the "pretty touches" requests) do not break the underlying business logic.

4. Design Concepts

4.1 Ergonomics and Aesthetics

We prioritized a "Premium" aesthetic to foster trust and engagement. Key design elements include:

  • Color Psychology: We used a palette dominated by Red (#b70b0b) to symbolize blood and urgency, complemented by dark blues and whites for professional contrast.
  • Fitts's Law: Action buttons (e.g., "Manage Donations", "Send Alert") are large and placed in easily accessible grid layouts to reduce acquisition time.
  • Consistency: All dashboards (Agent, Manager, Admin) share a common layout structure (Sidebar Navigation, Top Header, Grid Content), reducing the learning curve.

4.2 Visual Hierarchy

Using the Gestalt principle of proximity, related items are grouped in cards (e.g., "Quick Actions"). Gradient headers draw attention to page titles, and distinct styles separate interactive elements from static data.

5. Realization

This section highlights the key interfaces developed during the project.

5.1 The Dashboards

The dashboard is the central hub. We implemented a responsive grid layout that adapts to screen size. Features include:

  • Quick Actions Grid: Immediate access to high-frequency tasks.
  • Live Statistics: "Small Cards" displaying real-time data like total donors or stock levels.

5.2 Data Management (Tables)

Tables for "Universal Donors" and "Affiliated Donors" were designed for readability. We implemented styled headers with clear separation to prevent the "cramped" look, ensuring data is easy to scan. Sidebar navigation automatically highlights the active page, providing constant orientation feedback to the user.

5.3 Security & Recovery

We implemented secure authentication flows, including a redesigned "Reset Password" page. This page features a full-screen immersive background with a glass-morphism card effect, ensuring that even utility pages maintain the high aesthetic standard of the application.

6. Conclusion

The Blood Donation Management System project successfully demonstrates the application of HCI principles to a critical healthcare tool. By rigorously separating the architecture (MVC) and focusing on ergonomic design (Fitts's Law, Consistency), we created an interface that is both beautiful and highly functional.

Future improvements could include accessibility features (WCAG compliance) and a mobile-specific version for field agents.
